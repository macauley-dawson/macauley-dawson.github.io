Describe what the DOM is:

The Document Object Model, also known as the DOM, is a programming interface for web documents. It represents the webpage so that programmers can change the document structure, style, and content. There are many different aspects of the DOM, and that is why it is so handy to developers. You can look at what files/documents the current web page displayed has linked to it and what different types, such as HTML, CSS or JS files for example. Developers can give the webpage they are currently working on an audit on the web page which shows you how your webpage could function better, how it can be more accessible or to load quicker. The audit also shows you what youâ€™ve done well on your page too. The DOM allows developers to pull up specific information about a piece of code, fiddle around with it and delete bits if they want. An example of using the DOM for myself in the past couple weeks of this course at Enspiral Dev Academy was when I was first making this blog, I was using the DOM to add CSS style onto my HTML before actually making the changes stick in my code editor as it was an easy way for me to add, change and move different styles onto the HTML and seeing how it looks before pushing it up to go live.

Explain what functions are and why they are useful:

In Javascript, functions are groups of reusable code so that you don't have to write the same chunk of code out multiple times which saves you time and makes your code more efficient to read and reuse. They are also great for keeping your code nice and tidy, especially if you are using the same function over and over again such as a counter function, for example, on an online shopping site. Imagine the amount of code you would have to rewrite for every single product to have a counter list to allow the customers the option of getting more than just one of the same item?! Heaps, so functions are super handy in grouping code and making it reusable.
