Classes VS IDs

To start off understanding what a class is and what an ID is, we need to know the basics of creating a website. Firstly, there is the HTML. The HTML stands for HyperText Markup Language. It is the language in which you write the content of the website with. HTML has things called elements, which allow you to create and structure your website with.
Next off is the CSS. Once youâ€™ve got your content written up, you then use CSS to style it and make it look how you want it to look. CSS stands for Cascade Style Sheets, and allows you to change and modify the way your content in your HTML will look whilst viewing it in the browser.
Last but not least is Javascript. Java Script allows us to add even more changes to our HTML such as giving a page functionality and animation, to storing data from what the user is doing on the website.
Now, individually, these things do awesome jobs alone, but to get them to communicate to each other and work as one, you need Selectors. There are a few different selectors available to use but the main ones used are ID selector and the Class selector which is what ill be focusing on today.

What are the differences between Class and ID selectors?

The difference between an ID and a Class is that an ID is only used to identify one single component in our HTML. IDs are used when one element on the page should have a certain styling applied to it. However, a class can be used to target more than one component/element of HTML.

Best practices

When you work with CSS, there are no reasons making you use an ID over a class, but it is best practice to only use IDs if you want a style to apply to one element on the web page, and to use classes if you want a style to apply to multiple elements.
